Алгоритм (цикл) работы с Git 

(новый git репозиторий)
1. Инициализируем новый git репозиторий
git init
2. Связываем созданный репозиторий с удаленным 
облачным хранилищем (github.com)
git remote add origin https://github.com/Netsify/edu.kineu.kz.git
2.1 Статус файлов в локальном репозитории
git status
3. Вносим свои изменения в файлы
4. Знакомим измененные файлы с git
git add .
4.1 git status
5. Фиксируем изменения
git commit -m 'Название изменения (коммита)'
5.1 git status
On branch master
nothing to commit, working tree clean
6. Отправляем зафиксированные изменения в облако
git push origin master
6.1 git status
On branch master
nothing to commit, working tree clean

(существующий git репозиторий)
1. Клонируем git репозиторий
git clone https://github.com/Netsify/edu.kineu.kz.git НазваниеПапки
1.1 Текущие ветки проекта
git branch
2. Создаем ветку для работы и переходим на нее
git checkout -b НазваниеВетки
2.1 git branch
3. Вносим свои изменения в файлы
3.1 git status
4. Знакомим измененные файлы с git
git add .
5. Фиксируем изменения
git commit -m 'Название изменения (коммита)'
6. Заливаем проделанные изменения в основую ветку
git checkout master
git merge НазваниеВетки
7. Отправляем зафиксированные изменения в облако
git push origin master

(совместная работа)
1. Делаем fork (копию) существующего совместного git репозитория
через интерфейс github.com 
- Кнопка Fork в правом верхнем углу 
- выбираем себя, появляется копия репозитория для себя
- ссылка на копию - кнопка HTTPS clone URL в правом нижнем углу
2. Клонируем СВОЙ git репозиторий к себе на локальную машину
git clone https://github.com/Netsify/edu.kineu.kz.git НазваниеПапки
3. Вносим свои изменения в файлы
4. Знакомим измененные файлы с git
git add .
5. Фиксируем изменения
git commit -m 'Название изменения (коммита)'
6. Отправляем зафиксированные изменения в СВОЙ 
облачный git репозиторий
git push origin master
7. Создаем pull request (кнопка pull-request) через интерфейс github.com

Для получения всех принятых изменений
1. Создаем новую ветку и связываем ее с СОВМЕСТНЫМ git репозиторием
git checkout -b НазваниеВетки
git remote add НазваниеВетки https://github.com/Netsify/edu.kineu.kz.git
2. Закачиваем к себе последний принятые изменения
git pull НазваниеВетки master --rebase
3. Переходим на свою рабочую ветку и сливаем (merge) в нее закачанное
git checkout РабочаяВетка
git merge НазваниеВетки
4. git push origin НазваниеВетки (master)

git clean -d -f
